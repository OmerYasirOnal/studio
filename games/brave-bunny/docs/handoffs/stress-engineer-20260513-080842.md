## What was done

Stood up the PerfStress stress-test infrastructure for the Phase 5 perf contract
(60 fps / 200 enemies / 50 projectiles / 30 VFX on iPhone 12).

## Files updated

## Decisions / notes

- FPS floor is 30 (not 60) so the test passes on slow CI runners; iPhone 12 target remains 60 fps per contract.
- PerfStress.unity scene file is NOT hand-edited. The populator is the only mutation path.
- Enemy prefab `Assets/_Brave/Data/Definitions/Enemy_swarmer.prefab` is tried first; primitive cube tagged "Enemy" is the fallback.
- `FpsSampler` lives in `Brave.Diagnostics` namespace under `_Brave/Code/Diagnostics/` (not in Gameplay or Systems — diagnostic code is cross-cutting).
- No .asmdef was added for Diagnostics; the file is auto-referenced by the root assembly. A follow-up can add `Brave.Diagnostics.asmdef` if isolation is needed.
