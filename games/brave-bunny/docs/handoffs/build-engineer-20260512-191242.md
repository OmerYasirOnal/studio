## Why approach (a) over (b)

ProjectSettings.asset direct YAML edit was the alternative. Rejected because:
- The icon section uses `m_Textures: []` arrays referencing internal Unity GUIDs — a hand-written edit risks malformed YAML or wrong GUID linkage.
- An Editor script runs on every build, is reviewable C#, and is idempotent. Failure modes log warnings rather than corrupting a settings file.
- The brief's PostProcessBuild(50) note was reframed as "the safety-net's priority"; the **primary** wire-up is `IPreprocessBuildWithReport`, because `PlayerSettings.SetPlatformIcons` only takes effect BEFORE Unity's iOS build pass writes the xcassets — inline comment in the script explains this in detail.
