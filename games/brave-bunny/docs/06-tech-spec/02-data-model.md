# Tech Spec 02 — Data Model

> Owner: tech-architect. The ScriptableObject (SO) hierarchy used by every gameplay, systems, and UI subsystem in Brave Bunny. **Per repo-root `CLAUDE.md` principle 6, no magic numbers in scripts.** Everything tunable lives here as data, sourced from `data/balance/*.json` and generated by `core/tools/balance-tools/make_so_stubs.py`. Sister docs: `01-project-layout.md` (where SOs live), `03-save-system.md` (per-player state, not in SOs), GDD `03-characters.md` / `04-weapons.md` / `05-enemies.md` / `08-economy.md` (design source of truth).

## Principles

1. **SO = static, slot-shared, designer-facing config.** Save data is per-player and lives outside SOs (see `03-save-system.md`).
2. **Slugs are the join key.** Every SO has a `string slug` field; runtime lookups go through a `Catalog<T>` indexed by slug.
3. **Validation runs at OnValidate.** Bad data fails in the Editor, not at runtime.
4. **Polymorphism via script, not SO inheritance.** Concrete signature mechanics are derived C# classes referenced by name on the SO (see Polymorphic mechanics below) — keeps the dispatch compile-checked.
5. **Generated assets are committed.** The Python generator stamps a `// generated` header; manual edits break CI.

## Field-type legend

| Type | Meaning |
|---|---|
| `string slug` | kebab-case identifier matching GDD + `data/balance/*.json` |
| `LocalizedString` | wrapper around Unity Localization package key, defaults to English fallback |
| `Sprite` / `GameObject` | Unity asset reference |
| `float` | hit points, multipliers, seconds, world-units (units in field comments) |
| `int` | counts, currencies, levels |
| `enum` | exhaustive small set (Archetype, Role, Tier) |
| `T?` (nullable) | optional reference; null means "not applicable" |

## Top-level definition types

### CharacterDefinition

Mirrors GDD `03-characters.md`. One asset per character; vertical slice ships `bunny.asset` only.

```csharp
[CreateAssetMenu(menuName = "Brave/Character")]
public class CharacterDefinition : ScriptableObject {
    public string slug;                        // kebab-case, e.g. "bunny"
    public LocalizedString displayName;
    public Sprite portrait;                    // 256x256 UI portrait
    public GameObject prefab;                  // hero prefab incl. anim controller + collider
    public WeaponDefinition defaultStarterWeapon;  // ADR-0001: default only, not locked
    public CharacterStats baseStats;
    public string signatureMechanicTypeName;   // resolves to a SignatureMechanic subclass (see below)
    public ProgressionCurve levelCurve;        // XP-per-level for character meta levels 1..30
    public int unlockStarCost;                 // 0 for starter (Bunny)
    public AchievementDefinition? alternateUnlockAchievement;  // free unlock path

    void OnValidate() {
        if (string.IsNullOrEmpty(slug)) Debug.LogError($"{name}: slug required");
        if (baseStats == null) Debug.LogError($"{slug}: baseStats required");
    }
}

[Serializable]
public class CharacterStats {
    public float baseHP;            // hit points (Bunny baseline 100)
    public float baseMoveSpeed;     // units/second (Bunny baseline 4.5)
    public float damageMultiplier;  // multiplier vs weapon DMG (Bunny baseline 1.0)
    public float critRate;          // percent 0..100 (Bunny baseline 5)
    public float critDamage;        // multiplier (Bunny baseline 2.0)
    public float magnetMultiplier;  // multiplier on pickup magnet radius (Bunny baseline 1.0)
}

[Serializable]
public class ProgressionCurve {
    public int maxLevel;            // 30 for characters
    public AnimationCurve xpPerLevel;  // x = level (1..maxLevel), y = XP cost
}
```

Validation: `slug != null`, `prefab != null`, `levelCurve.maxLevel == 30`, `baseStats.baseHP > 0`.

### WeaponDefinition

Mirrors GDD `04-weapons.md`. 12 launch weapons; vertical slice ships 3.

```csharp
[CreateAssetMenu(menuName = "Brave/Weapon")]
public class WeaponDefinition : ScriptableObject {
    public string slug;
    public LocalizedString displayName;
    public WeaponArchetype archetype;          // enum
    public TargetingMode targeting;            // enum
    public WeaponLevelData[] levels;           // EXACTLY length 5 (L1..L5)
    public WeaponEvolutionRecipe? evolution;   // null for non-evolving weapons (6 of 12)
    public SynergyTag[] synergyTags;
    public Sprite icon;
    public GameObject projectilePrefab;        // can be null for aura/utility

    void OnValidate() {
        if (levels == null || levels.Length != 5)
            Debug.LogError($"{slug}: weapons must have exactly 5 levels");
    }
}

public enum WeaponArchetype { Projectile, Area, Aura, Summon, Utility }
public enum TargetingMode { Nearest, Furthest, Random, SelfCentered, OrbitPlayer, RandomScreenPos }
public enum SynergyTag { Kinetic, Nature, Solar, Frost, Aura, Summon, Mech, Explosive, Bounce, Beam, Heavy }

[Serializable]
public class WeaponLevelData {
    public float damage;          // DMG per hit
    public float fireRate;        // seconds between fires
    public float range;           // world units
    public int projectiles;       // count per fire
    public string upgradeFlavor;  // 1-line designer description for the draft card
}

[Serializable]
public class WeaponEvolutionRecipe {
    public PassiveDefinition ingredient;       // L5 passive required
    public WeaponDefinition resultWeapon;      // evolved form
}
```

### EnemyDefinition

Mirrors GDD `05-enemies.md`. Roles + biome variants.

```csharp
[CreateAssetMenu(menuName = "Brave/Enemy")]
public class EnemyDefinition : ScriptableObject {
    public string slug;
    public EnemyRole role;                     // Swarmer, Tank, Ranged, Elite
    public BiomeDefinition biome;              // null = any biome (rare)
    public GameObject prefab;
    public float baseHP;                       // minute-1 baseline; scaling lives on biome
    public float contactDamage;                // hp/touch
    public float? rangedDamage;                // null if not ranged
    public float moveSpeed;                    // units/sec (NOT a multiplier here; raw)
    public DropTable drops;
    public LocalizedString telegraphSfxKey;    // pre-attack audio
    public float telegraphWindowSeconds;       // 0 for swarmers

        if (role != EnemyRole.Swarmer && telegraphWindowSeconds <= 0)
            Debug.LogWarning($"{slug}: non-swarmer should have a telegraph window");
    }
}

public enum EnemyRole { Swarmer, Tank, Ranged, Elite }

[Serializable]
public class DropTable {
    public float xpGemSmallChance;    // 0..1
    public float xpGemMediumChance;
    public float xpGemLargeChance;
    public float goldCoinChance;
    public float heartChance;
    public int soulShardsOnKill;      // 0 for trash, 1 for elite, 3-30 for bosses
}
```

### WaveDefinition

Loaded from `data/waves/<biome>.json` (level-designer's source of truth per `brave-bunny/CLAUDE.md`). The SO is a thin wrapper around the parsed schedule so gameplay-engineer's wave driver can subscribe.

```csharp
[CreateAssetMenu(menuName = "Brave/Wave")]
public class WaveDefinition : ScriptableObject {
    public string biomeSlug;
    public WaveEvent[] events;                 // ordered by triggerMinute

    void OnValidate() {
        if (events != null) {
            for (int i = 1; i < events.Length; i++)
                if (events[i].triggerMinute < events[i-1].triggerMinute)
                    Debug.LogError($"{biomeSlug}: wave events must be sorted by triggerMinute");
        }
    }
}

[Serializable]
public class WaveEvent {
    public float triggerMinute;       // 0..10+
    public WaveEventType type;        // Spawn, MiniBoss, Boss, Lull
    public EnemyDefinition enemy;     // for Spawn
    public int spawnCount;
    public SpawnPattern pattern;      // Ring, Arc, RandomEdge, ScriptedPoints
}

public enum WaveEventType { Spawn, MiniBoss, Boss, Lull }
public enum SpawnPattern { Ring, Arc, RandomEdge, ScriptedPoints }
```

## Polymorphic mechanics: SignatureMechanic + AchievementCondition + BossAttackPattern

These three subsystems share a pattern: data references a string type name; runtime resolves it to a concrete C# class. We **do not** store the polymorphic behavior on the SO as `[SerializeReference]` because (a) it's brittle across renames, (b) we want each mechanic to be exhaustively listed in code for code review, and (c) signature mechanics are not data — they are gameplay code with shared interfaces.

```csharp
public abstract class SignatureMechanic {
    public abstract string TypeName { get; }
    public abstract void OnAttach(PlayerContext ctx);
    public abstract void Tick(PlayerContext ctx, float dt);
    public abstract void OnDetach(PlayerContext ctx);
}

// One concrete subclass per character. Registered in a SignatureMechanicRegistry at boot.
public sealed class HopDodgeMechanic : SignatureMechanic {
    public override string TypeName => "hop-dodge";
    // ...
}
public sealed class ShellBraceMechanic : SignatureMechanic { ... }
public sealed class ThornRingMechanic : SignatureMechanic { ... }
public sealed class CunningStrikeMechanic : SignatureMechanic { ... }
public sealed class SplashVolleyMechanic : SignatureMechanic { ... }
public sealed class HeartySnackMechanic : SignatureMechanic { ... }
public sealed class BabyPatrolMechanic : SignatureMechanic { ... }
public sealed class FarSightMechanic : SignatureMechanic { ... }
```

`CharacterDefinition.signatureMechanicTypeName` resolves via the registry at character spawn. A `[Test]` in `Brave.Tests.EditMode` walks every `CharacterDefinition` asset and asserts the type name resolves — broken references fail CI, not runtime.

The same pattern applies to `AchievementCondition` (e.g., "kill 1000 swarmers in a run") and `BossAttackPattern` (one class per boss phase pattern).

## Catalogs and lookup

At boot, `Brave.Boot.Bootstrap` scans `Assets/_Brave/Data/` (via Addressables labels or `Resources.LoadAll<T>`) and builds:

```csharp
public class Catalog<T> where T : ScriptableObject {
    private Dictionary<string, T> _bySlug;
    public T Get(string slug) => _bySlug[slug];
    public bool TryGet(string slug, out T def) => _bySlug.TryGetValue(slug, out def);
    public IReadOnlyList<T> All => _bySlug.Values.ToList();
}
```

Catalogs are passed via constructor injection to systems (gameplay's `RunController`, ui's `LoadoutScreen`, etc.). No singletons; no `Resources.Load` at runtime in hot paths.

## JSON → ScriptableObject generation

The Python tool `core/tools/balance-tools/make_so_stubs.py` converts:

| Source JSON | Target SO type | Target folder |
|---|---|---|
| `data/balance/characters.json` | `CharacterDefinition` | `Assets/_Brave/Data/Balance/Characters/` |
| `data/balance/weapons.json` | `WeaponDefinition` | `Assets/_Brave/Data/Balance/Weapons/` |
| `data/balance/enemies.json` | `EnemyDefinition` | `Assets/_Brave/Data/Balance/Enemies/` |
| `data/balance/bosses.json` | `BossDefinition` | `Assets/_Brave/Data/Balance/Bosses/` |
| `data/balance/scaling.json` | feeds `BiomeDefinition.hpScaling` | `Assets/_Brave/Data/Balance/Biomes/` |
| `data/waves/<biome>.json` | `WaveDefinition` | `Assets/_Brave/Data/Balance/Waves/` |
| `data/balance/passives.json` | `PassiveDefinition` | `Assets/_Brave/Data/Balance/Passives/` |
| `data/balance/economy.json` | feeds `UpgradeNodeDefinition`, `CosmeticDefinition` prices | `Assets/_Brave/Data/Balance/UpgradeNodes/`, `.../Cosmetics/` |
| `data/balance/achievements.json` | `AchievementDefinition` | `Assets/_Brave/Data/Definitions/Achievements/` |
| `data/balance/battlepass.json` | `BattlePassSeasonDefinition` + tiers | `Assets/_Brave/Data/Balance/BattlePass/` |

Each generated `.asset` file starts with a YAML comment header:

```yaml
%YAML 1.1
%TAG !u! tag:unity3d.com,2011:
# generated by make_so_stubs.py — do not edit by hand
```

CI fails any commit where a generated file's content drifts from a fresh regeneration.
