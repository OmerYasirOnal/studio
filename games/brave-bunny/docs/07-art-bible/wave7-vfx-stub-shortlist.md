## Purpose

Wave 7 wired `VfxPool` (Brave.Gameplay.Pooling) and `PooledVfx`
(ParticleSystem-event-driven auto-release per ADR-0019). **No VFX prefabs
exist yet** under `unity/Assets/_Brave/Art/Vfx/` — code paths that would call
`VfxPool.Play(...)` are stubbed (`// qa-engineer adds VFX hit-flash here`
in `EnemyHealth.cs`) and silently no-op until prefabs land.

This file is the art-director pickup list for the **20 vertical-slice VFX
effects** enumerated in `05-vfx-style.md` § VFX checklist for vertical slice.

## Sourcing approach (CC0 only)

Per `core/docs/asset-policy.md`, no paid asset packs. The VFX strategy is:

1. **Unity built-in templates** (CC0 / Unity Companion License, free, ships with the editor):
   - `Particles/Default-Particle` material
   - `GameObject → Effects → Particle System` (Shuriken / VFX Graph defaults)
2. **Recolor + retime** per `01-color-palette.md` accent palette and the timing register in `05-vfx-style.md`.
3. **No Shuriken CPU sim** except boss arenas — port to **URP VFX Graph** with the toon-tinted quad shader from `_Brave/Shaders/`.
4. **Sprite-sheet flipbooks** (optional accent) — pull from **Kenney Particle Pack (CC0)** if a frame-based puff is needed for the daisy explode or pollen drift.

No third-party VFX libraries beyond Kenney's CC0 packs.

## Performance reminder (cross-checked with `games/brave-bunny/CLAUDE.md`)

- Per `05-vfx-style.md`: combined VFX budget 3.5 ms / frame (5 ms total − 1.5 ms post).
- Each prefab MUST have `ParticleSystem.main.stopAction = Callback` so `PooledVfx.OnParticleSystemStopped` fires the pool-return (per ADR-0019). Else the pool emits a one-time fallback-timeout warning.
- GPU-instanced quads only outside boss arenas — author with VFX Graph, not CPU Shuriken.

## Hand-off

- Asset-curator + art-director author prefabs under the `_Brave/Art/Vfx/<bucket>/` paths above.
- Each prefab uses the `PooledVfx` MonoBehaviour wired by the prefab's root; particle reference set via `Reset()`/`Awake()`.
- Once prefabs land, `VfxPool` instances are registered at Boot keyed by their prefab basename; gameplay code calls `VfxPool.Play(key, position)`.
- Status: **20 prefabs outstanding** as of Wave 7B → soft-launch-prep. No prefab currently exists in `_Brave/Art/Vfx/`.
- No CC0-license attribution required for Unity built-in PS templates or Kenney Particle Pack.
